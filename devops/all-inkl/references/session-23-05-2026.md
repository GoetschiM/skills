## Pitfalls Found

### 1. Parameter nesting
`kas_mail` create requires `params` OBJECT inside `arguments`, NOT flat:
- ✅ `"arguments":{"action":"create","params":{"local_part":"hermes","domain_part":"radislione.net","mail_password":"ApolloHermes2026!"}}`
- ❌ `"arguments":{"action":"create","local_part":"hermes","domain_part":"radislione.net","mail_password":"ApolloHermes2026!"}`

### 2. Stdin input method
`subprocess.run(input=payload)` with mcp-all-inkl returns EMPTY stdout.
**Must use heredoc** or pipe via `subprocess.Popen` with `stdin=subprocess.PIPE`.
Shell heredoc works perfectly.

### 3. env vs export
`export KAS_LOGIN="w019000a"` then `npx` command — env vars sometimes not picked up.
**Always inline:** `KAS_LOGIN="w019000a" KAS_PASSWORD="***" npx -y mcp-all-inkl`
Or use `env KAS_LOGIN=... KAS_PASSWORD=... npx -y mcp-all-inkl`

### 4. Response format
Always `{"result":{"content":[{"type":"text","text":"<actual-data>"}]},"jsonrpc":"2.0","id":N}`
Extract with: `json.loads(result)["result"]["content"][0]["text"]`

### 5. Tools list
```json
{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}
```
Returns all 9 tools with input schemas.
