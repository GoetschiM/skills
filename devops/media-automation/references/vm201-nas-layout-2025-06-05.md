## Critical: Two Movie Directories

| Path | Managed by | Status |
|------|-----------|--------|
| `/media/NAS/Movie/` (singular) | ❌ Manuell, alt | Het Dateie, aber **nöd** via Radarr |
| `/media/NAS/Movies/` (plural) | ✅ Radarr-Ziel | **Leer** — Root-Folder muess da dra zeige |

**Radarr root folder z'konfiguriere:** `/media/NAS/Movies/` (PLURAL), NICHT `/media/NAS/Movie/`.

## Container Volume Mappings

| Container | Host Path | Container Path |
|-----------|-----------|----------------|
| qBittorrent | `/media/NAS/Downloads` | `/downloads` |
| Radarr | (config: `/DATA/AppData/radarr/config`) | `/config` |
| Sonarr | (config: `/DATA/AppData/sonarr/config`) | `/config` |

Radarr/Sonarr verwennde **kei host-mounts für media** — si schriended via Docker internal paths (mapped über docker-compose volumes).

## Discovery Method (for future sessions)

Diese Informatione chönd über SSH via VM 201 abgruefe werde:

```bash
# SSH (Key via CasaOS File API injiziert)
ssh root@10.0.60.201

# NAS directory layout
ls -la /media/NAS/

# Check exFAT
df -T /media/NAS/

# Disk usage
df -h /media/NAS/
```
