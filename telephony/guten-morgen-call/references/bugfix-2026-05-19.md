# Bugfix: Guten-Morgen-Call Script (19.05.2026)

## Drei kritische Bugs gefunden und behoben

Im Rahmen der TTS-Voice-Umstellung wurden drei Bugs im Script
`scripts/guten-morgen-call.py` entdeckt, die alle den ausgehenden
Anruf betrafen.

### Bug 1: Falsche TTS-Voice (VOICE-Konstante)

**Problem:** `VOICE = "de-DE-FlorianMultilingualNeural"`  
**Korrektur:** `VOICE = "de-DE-KillianNeural"`

Der Fehler: Das Script hatte Apollos Stimme (Florian) hartcodiert,
statt Hermes' Stimme (Killian) zu verwenden. Zudem war Florian die
falsche Stimme für einen Hermes-Call.

**Pitfall:** Beim Kopieren/Klonen von Call-Skripten die `VOICE`-Konstante
immer auf den richtigen Agent abgleichen. Jeder Agent braucht seine eigene
Stimme.

### Bug 2: Falsches Rufnummern-Format (PHONE_NUM)

**Problem:** `PHONE_NUM = "+41796459743"`  
**Korrektur:** `PHONE_NUM = "0796459743"`

Der Fehler: Der `[apollo-out]` Dialplan in Asterisk erwartet Swiss-Format
(079...), stripped die erste Ziffer und ersetzt mit +41. Bei Eingabe
von +41... funktioniert der Dialplan nicht korrekt.

**Pitfall:** Immer Swiss-Format ohne +41 verwenden. Das `+41` wird
vom Dialplan `Dial(PJSIP/salt-trunk/sip:+41${EXTEN:1}@sipvoice.salt.ch)`
automatisch hinzugefügt.

### Bug 3: Falsche Extension (s@default statt s@apollo-external)

**Problem:** `extension s@default`  
**Korrektur:** `extension s@apollo-external`

Der Fehler: `s@default` spielt die Datei `demo-congrats` ab statt
der TTS-Ansage. Der korrekte Post-Answer-Context ist `s@apollo-external`,
der `Playback(apollo_notify)` ausführt.

**Pitfall:** Vor jedem Einsatz prüfen: `grep 'extension s@' scripts/guten-morgen-call.py`
Muss `s@apollo-external` sein, nicht `s@default`.

### Zusätzlich: Falscher Sign-Off im Briefing-Text

**Problem:** Sign-off sagte "dein Apollo!"  
**Korrektur:** "dein Hermes!"

Das ist kein technischer Bug, aber verwirrend für den Empfänger.
