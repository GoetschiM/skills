## Nova Model Fix (gemma4)

### Problem
Nova downloaded `gemma4:e4b` (8B params, 9.6GB) and set it as default.
But config had `context_length: 8192` → Hermes requires min 64K:
```
ValueError: Model gemma4:e4b has a context window of 8,192 tokens,
which is below the minimum 64,000
```
Nova was "dead" after restart.

### Root Cause
Config hardcoded `context_length: 8192` but the model SUPPORTS 131,072 natively.

### Fix 1: Set context_length to 131072
Initial fix: edit context_length from 8192 to 131072 via YAML parser.
⚠️ yaml.dump() reorders all keys alphabetically → file looks different but works.
Gateway: `systemctl restart hermes167.service`

### Fix 2: Switch to smaller model
User said 8B too big for Nova's 14GB RAM (only 3.6GB free).

Attempted:
1. `gemma4:e2b-mlx` → ❌ macOS only (requires Apple MLX)
2. `gemma4:e2b-it-q4_K_M` → ✅ 5.1B params, Q4_K_M, 7.2GB, 131K context

Updated config:
```yaml
model:
  default: gemma4:e2b-it-q4_K_M
  provider: ollama-local
providers:
  ollama-local:
    default_model: gemma4:e2b-it-q4_K_M
    models:
      gemma4:e2b-it-q4_K_M:
        context_length: 131072
```

### How to Check Nova Config

ssh root@10.0.60.167 'cat /opt/hermes167/data/config.yaml'

Restart Nova: `systemctl restart hermes167.service`
Check logs: `journalctl -u hermes167.service -n 30`
