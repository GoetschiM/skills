# Telethon Setup — 14. Mai 2026

## Kontext
Erstmaliger Login von Michels Telegram-Account via Telethon, nach dem der `telegram-telethon` Skill bereits existierte aber die Session nicht authentifiziert war.

## Ist-Zustand vor Setup
- `/opt/data/home/.hermes/telegram_config.json` existierte — aber OHNE `"phone"`-Feld
- `/opt/data/home/.hermes/michel_telethon.session` existierte (28KB) — aber **nicht authorisiert**
- `setup.py` hatte `await client.start()` ohne Args → würde input() brauchen → EOFError

## Durchgeführter Ablauf

### 1. Config reparieren
Config fehlte `phone`-Feld. Manuell ergänzt:
```json
{
  "api_id": "28054408",
  "api_hash": "7e7b6ab2dd21a99af5ff0d68b550045e",
  "phone": "+41796459743",
  "my_user_id": 322663922,
  "my_name": "Michel"
}
```

### 2. Code-Anfrage senden (separates Script)
`/tmp/send_telegram_code.py`:
```python
client = TelegramClient(SESSION, int(CONFIG["api_id"]), CONFIG["api_hash"])
await client.connect()
await client.send_code_request(phone)
print("CODE_SENT")
```
→ Michel bekam den Code auf sein Handy.

### 3. Login mit --code Flag
```bash
python3 setup.py --code 85081
```
→ **Erfolg:** `✅ Eingeloggt als: Michel (@Radislione)`

### 4. Verifikation
Nach Login `get_dialogs()` → 10 Chats sichtbar, inkl. DMs mit Henry, MX5 Bots, Peanut Paprika, Nanoclaw, etc.

## Fehler, die auftraten

| Fehler | Ursache | Lösung |
|--------|---------|--------|
| `EOFError: EOF when reading a line` | `input()` in non-interactive terminal | `--code` Flag verwenden |
| `input()` für Telefonnummer | Config hatte kein `phone`-Feld | Config manuell erweitern |
| Script löscht alte `.session` | `client.start()` bei existenter aber unauthlicher Session | `send_code_request()` + `sign_in()` separat |
